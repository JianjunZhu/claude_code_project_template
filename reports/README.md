# reports/ 每轮研究报告目录

> 本目录存放**研究迭代每一轮产生的报告**（对应 [`../docs/rules/RESEARCH_LOOP.md`](../docs/rules/RESEARCH_LOOP.md) 环 2 的"⑤ 形成研究报告"）。本模板于 2026-06-19 创建，目录内无真实报告，下文为占位约定。

## 1. 定位（与其他目录区分）

- `reports/`：把一轮的多次运行**综合**成"本轮确立了什么 / 证据如何 / 是否达标 / 下一步"的叙事报告。
- 区别：
  - [`../experiments/records/`](../experiments/records/) 是**单次运行**的操作记录；
  - [`../results/`](../results/) 是**结果数据 / 表 / 图**；
  - `reports/` 是**高层综合**，引用上述两者。

## 2. 命名与内容

- 活跃报告：`reports/round-<NN>_<主题>.md`（NN 为轮次编号）。
- 每份报告建议含：本轮目标 / 做了什么 / 关键结果（链接 `results`、`outputs`）/ 证据等级 / 结论是否达成科研目标 / 暴露的 gap / 下一步。

## 3. 定期归档总结（报告过多时）

- 当活跃报告积累过多（如 > 8 份，或一个阶段结束）时，把较旧报告**移入** `reports/archive/`，并在 [`SUMMARY.md`](SUMMARY.md) 中**滚动汇总**其要点（结论 + 证据等级 + 链接）。
- **只归档、不删除**：正文进 `archive/`，要点留 `SUMMARY.md`，便于回溯而不淹没活跃区；负结果报告同样保留。
