# src/ 项目源码目录

> 本目录存放**本项目自有源码**（库 / 模块 / 模型 / 数据处理逻辑）。本模板于 2026-06-19 创建，目录内尚无真实代码，下文为占位约定。源码与轻量文件入 git。

## 1. 定位与边界

- `src/`：可被复用、可被 import 的**自有源码**（模型定义、数据加载、训练 / 评估逻辑、工具函数）。
- 与其他目录区分：
  - 运行入口脚本（`train_` / `eval_` / `prep_` / `plot_`）放 [`../scripts/`](../scripts/)，调用 `src/` 的逻辑。
  - 第三方 / 下载的代码放 [`../third_party/`](../third_party/)，**不要**混进 `src/`。
  - 配置放 [`../configs/`](../configs/)；不在源码里硬编码路径 / 超参（用 `<数据路径>`、`<输出目录>` 等占位符或配置注入）。

## 2. 组织建议

- 按职责分模块 / 包，如 `src/<包名>/{data,models,train,eval,utils}/`。
- 不硬编码私有绝对路径 / 密钥 / token；路径与超参经 `configs/` 注入。
- 关键逻辑（指标计算、数据划分、评估）应有测试，避免静默 bug 污染结果。

## 3. 可复现与纪律

- 显式设置并记录 seed；运行行为由 `configs/experiments/` 的配置控制。
- 修改影响结果 / 结论的源码时，同步更新对应文档（见 `../CLAUDE.md` 第 9 / 12 节）。
- 删除源码属不可逆操作，需明确指令，优先归档。
