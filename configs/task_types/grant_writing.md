# 任务类型配置：grant_writing（基金申请书写作）

> 配置可选，基础模板不依赖任何单一任务类型。
> 本文件仅在当前项目确属"基金申请书写作"任务时启用；启用方式为在 `docs/records/PROJECT.md` 标注任务类型并按需引用本文件。所有具体取值用占位符表达，未知=TBD、未验证=PENDING、未确认=NOT VERIFIED、未完成=INCOMPLETE。

## 1. 任务目标

- 研究目标：`<研究目标>`（TBD），基金类型：`<基金项目类型>`（TBD）。
- 任务形式：撰写基金申请书，核心是清晰区分"已完成/在研/拟开展"三类工作，并为各部分匹配恰当证据强度。
- 任务边界：明确申请方向、研究内容边界、预期成果与可行性论证范围（TBD）。
- 评价口径：以工作阶段标注是否准确、预研证据是否可追溯为口径。
- 未验证结论标 PENDING/NOT VERIFIED，拟开展工作不得描述为已有结果。

## 2. 默认交付物

- 申请书各章节草稿（研究背景、内容、方案、可行性、基础，路径 TBD）。
- 工作阶段标注表：每项工作标注"已完成/在研/拟开展"。
- 预研证据清单：已完成部分链接 `docs/records/EVIDENCE.md` 与 artifact。
- 研究计划与里程碑（拟开展部分，时间线 TBD）。
- 参考文献与前期成果清单。
- 实验记录：`experiments/records/` 中作为预研依据的来源。

## 3. 推荐指标

> 基金写作以"阶段标注准确性与证据匹配"为核心，不填具体科研数值。

- 阶段标注完备性：每项工作均标注已完成/在研/拟开展。
- 预研证据可追溯性：已完成部分均有 artifact 与等级（1→9）。
- 声称与证据匹配度：已完成部分不夸大，拟开展部分不当作既得结果。
- 可行性论证充分性：方案、风险与备选路径是否说明（取值 TBD/PENDING）。

## 4. 常见失败模式

- 阶段混淆：把拟开展工作写成已完成结果，或把初步预研夸大为成熟结论。
- 过度声称：用低等级证据支撑"已实现/已验证"表述。
- 预研不可追溯：缺 artifact、commit hash 或来源记录。
- 可行性论证薄弱：无风险分析与备选方案。
- 里程碑与工作量不匹配，时间线不现实。
- 把 PENDING/NOT VERIFIED 结论当作既得成果列入研究基础。

## 5. 证据要求

引用统一证据等级（1→9）。

- 每项"已完成"工作须可回答：是什么 / 哪个 artifact / 在哪 / 何时 / 何命令 / 等级是几。
- 申请书中报告的等级不得高于证据实际等级；拟开展工作用"我们假设/预期/拟验证"等谨慎措辞。
- 默认状态：未核验预研数字标 PENDING，未确认结论标 NOT VERIFIED。
- 负结果与风险须在可行性分析中如实呈现。
- 不得声称"临床有效/泛化强/鲁棒/SOTA/可临床使用"，除非证据与措辞匹配。

## 6. 优先查看的文件

1. `docs/records/EVIDENCE.md`（预研证据来源，最优先）。
2. `docs/records/PROJECT.md` 与 `docs/records/TASK_BRIEF.md`（项目事实与方向）。
3. `docs/records/EXPERIMENT_LOG.md`（前期工作记录）。
4. `docs/records/RESULT_AUDIT.md`（预研数字核验）。
5. `docs/rules/RESEARCH_RULES.md`（科研安全表达纪律）。
6. `CLAUDE.md`（Agent 行为规则）。

## 7. 典型输出

- 申请书章节草稿（含阶段标注，结论状态 TBD/PENDING/NOT VERIFIED）。
- 工作阶段标注表（已完成/在研/拟开展）。
- 预研证据清单（链接 artifact 与等级）。
- 研究计划、里程碑与可行性/风险分析。
- 前期成果与参考文献清单。
