# 任务类型配置：reinforcement_learning（强化学习）

> 配置可选，基础模板不依赖任何单一任务类型。
> 本文件仅在当前项目确属"强化学习"任务时启用；启用方式为在 `docs/PROJECT.md` 标注任务类型并按需引用本文件。所有具体取值用占位符表达，未知=TBD、未验证=PENDING、未确认=NOT VERIFIED、未完成=INCOMPLETE。

## 1. 任务目标

- 研究目标：`<研究目标>`（TBD）。
- 任务形式：在 `<数据集名称>`/环境中通过与环境交互学习策略以最大化长期回报，输出策略并评估其回报、成功率与样本效率。
- 任务边界：明确环境、观测/动作空间、奖励定义、是否离线 RL（TBD）。
- 评价口径：按 episode 评估，区分训练曲线与冻结评估；跨种子统计。
- 未验证结论标 PENDING/NOT VERIFIED。

## 2. 默认交付物

- 训练/评估脚本：`<主脚本>`（TBD）。
- 配置文件：`configs/`（seed、环境配置、算法超参、奖励设定，TBD）。
- 策略/checkpoint：`<模型权重路径>`（默认不入 git，manifest+checksum）。

## 3. 推荐指标

> 取值留占位，不得填未经真实 artifact 支持的数字。

- 主要指标：平均 return（episodic return）、任务成功率（`<主要指标>`=TBD）。
- 样本效率：达到目标性能所需交互步数/样本数。
- 稳定性：跨种子 return 分布、训练曲线方差。
- 辅助指标：归一化得分、最终策略熵（`<辅助指标>`=TBD）。
- 聚合方式：跨种子统计需固定并记录，报告均值与置信区间/方差。

折扣回报示意：

$$
G_t = \sum_{k=0}^{\infty} \gamma^{k}\, r_{t+k+1}
$$

## 4. 常见失败模式

- 种子敏感：少数种子表现好被当作普遍结论，方差未报告。
- 评估与训练混淆：用训练曲线峰值而非冻结评估报告性能。
- 奖励塑形导致目标偏移，刷高 return 但行为不符合任务意图。
- 离线 RL 数据泄漏或分布外评估失真。
- 样本效率口径不一致（环境步 vs 梯度步）致不可比。
- 在评估配置上反复调超参，泛化被高估。

## 5. 证据要求

引用统一证据等级（1→9）。

- 每个 RL 指标结论须可回答：结论是什么 / 哪个 artifact / 在哪 / 何时 / 何命令 / 初步还是最终。
- 默认状态：训练 INCOMPLETE、指标 PENDING、对外结论 NOT VERIFIED。
- 报告多种子结果与方差，不以单种子下结论；区分训练与冻结评估。
- 测试前冻结环境版本、种子集合、评估脚本；不在评估配置上调参。
- 保留失败种子/不收敛运行等负结果。
- 证据不足不得声称"更高效/更稳/更通用/SOTA"，用谨慎措辞。
