# 任务类型配置：robotics（机器人）

> 配置可选，基础模板不依赖任何单一任务类型。
> 本文件仅在当前项目确属"机器人"任务时启用；启用方式为在 `docs/records/PROJECT.md` 标注任务类型并按需引用本文件。所有具体取值用占位符表达，未知=TBD、未验证=PENDING、未确认=NOT VERIFIED、未完成=INCOMPLETE。

## 1. 任务目标

- 研究目标：`<研究目标>`（TBD）。
- 任务形式：在 `<数据集名称>`/仿真或真机环境中完成指定操作/导航/控制任务，输出策略或控制器并评估其完成质量与安全性。
- 任务边界：明确任务集合、观测/动作空间、仿真与真机差异、安全约束（TBD）。
- 评价口径：按 episode 评估，区分仿真与真机；先算单 episode 后 aggregate。
- 未验证结论标 PENDING/NOT VERIFIED。

## 2. 默认交付物

- 训练/部署脚本：`<主脚本>`（TBD）。
- 配置文件：`configs/`（seed、环境/任务配置、控制频率、超参，TBD）。
- 策略/控制器：`<模型权重路径>`（默认不入 git，manifest+checksum）。

## 3. 推荐指标

> 取值留占位，不得填未经真实 artifact 支持的数字。

- 主要指标：任务成功率（`<主要指标>`=TBD）。
- 安全指标：安全违例次数/比例、碰撞/越界次数。
- 效率指标：完成时间、路径长度、能耗（辅助，`<辅助指标>`=TBD）。
- 鲁棒性：跨随机种子与扰动条件的成功率分布。
- 聚合方式：跨 episode/种子统计需固定并记录，报告均值与方差。

## 4. 常见失败模式

- 仿真到真机差距（sim-to-real gap）：仿真高成功率不代表真机可用。
- 安全违例被成功率掩盖：只看成功率忽略碰撞/越界。
- 评估种子过少，成功率方差大、结论不稳。
- 奖励/任务定义与真实目标不一致，刷指标但行为不合理。
- 数据/场景泄漏：训练与评估共用相同布局/物体实例。
- 在评估场景上反复调参导致高估泛化。

## 5. 证据要求

引用统一证据等级（1→9）。

- 每个机器人指标结论须可回答：结论是什么 / 哪个 artifact / 在哪 / 何时 / 何命令 / 初步还是最终。
- 默认状态：训练/部署 INCOMPLETE、指标 PENDING、对外结论 NOT VERIFIED。
- 区分仿真与真机证据等级，不得用仿真结果声称真机可用。
- 成功率与安全违例须同时报告；保留失败/违例 episode 等负结果。
- 测试前冻结环境版本、种子集合、评估脚本；不在评估场景上调参。
- 证据不足不得声称"更安全/更可靠/可部署/SOTA"，用谨慎措辞。

## 7. 典型输出

- 评估轨迹与日志（默认不入 git，manifest+checksum 记录）。
- 成功率与安全违例汇总表、跨种子分布（取值 TBD/PENDING）。

- 评估运行 manifest：commit hash、环境版本、seed 集合、命令、参数、时间。
- 实验记录条目，标注仿真/真机来源、证据等级与状态。
