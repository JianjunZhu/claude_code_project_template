# 任务类型配置：segmentation（分割）

> 配置可选，基础模板不依赖任何单一任务类型。
> 本文件仅在当前项目确属"分割"任务时启用；启用方式为在 `docs/records/PROJECT.md` 标注任务类型并按需引用本文件。所有具体取值用占位符表达，未知=TBD、未验证=PENDING、未确认=NOT VERIFIED、未完成=INCOMPLETE。

## 1. 任务目标

- 研究目标：`<研究目标>`（TBD）。
- 任务形式：对 `<数据集名称>` 中的目标区域逐像素/逐体素预测类别或掩膜，输出与输入空间对齐的分割结果。
- 任务边界：明确分割对象（前景类别清单 TBD）、是否多类、是否实例级、维度（2D/3D，TBD）。
- 评价口径：先在 case-level 计算，再按协议 aggregate；区分开发集与冻结测试集结果，不混用。
- 本任务的所有结论默认证据等级见各结论旁标注；未验证结论标 PENDING/NOT VERIFIED。

## 2. 默认交付物

- 训练/推理脚本：`<主脚本>`（路径 TBD）。
- 配置文件：`configs/` 下对应实验配置（含 seed、数据划分、超参，TBD）。
- 模型权重：`<模型权重路径>`（默认不入 git，用 manifest+checksum 记录）。
- 分割结果输出：`<输出目录>` 下掩膜文件 + 评估表（默认不入 git）。
- 评估脚本与结果表：固定评估脚本 + 指标 manifest（记录命令、commit hash、生成时间）。
- 实验记录：`experiments/records/` 下逐次运行记录。

## 3. 推荐指标

> 取值一律留占位，不得填入未经真实 artifact 支持的数字。

- 主要指标：Dice 系数（`<主要指标>`=TBD）。
- 重叠指标：IoU（Jaccard）。
- 边界指标：HD95（95% Hausdorff 距离）、ASSD（平均对称表面距离）。
- 辅助指标：召回率、精确率、体积误差（`<辅助指标>`=TBD）。
- 聚合方式：先 case-level 后 aggregate（均值/中位数/分位数），需说明是否按类加权。
- 指标定义、版本与实现需固定并记录，跨实验保持一致。

Dice 定义（示意）：

$$
\mathrm{Dice} = \frac{2\,|P \cap G|}{|P| + |G|}
$$

其中 $P$ 为预测掩膜，$G$ 为标注掩膜。

## 4. 常见失败模式

- 标注质量问题：边界不一致、缺标、类别定义漂移。
- 类别不平衡：小目标/稀有类被主导类淹没，Dice 虚高。
- 数据泄漏：同一受试者/序列同时落入 train 与 test；预处理统计量用到了测试数据。
- 评估口径不一致：是否含背景类、是否后处理、重采样到不同分辨率导致指标不可比。
- 空预测陷阱：对无目标 case 的 Dice 处理约定不统一。
- 过拟合开发集：反复在同一验证集调阈值/后处理，泛化被高估。

## 5. 证据要求

引用统一证据等级（1 假设 → 2 设计方案 → 3 实现正确性检查 → 4 开发集验证 → 5 内部验证 → 6 冻结内部测试 → 7 锁定 holdout 测试 → 8 外部验证 → 9 独立复现）。

- 每个分割指标结论须可回答：结论是什么 / 由哪个 artifact 支持 / artifact 在哪 / 何时生成 / 由什么命令生成 / 初步还是最终。
- 当前默认状态：训练 INCOMPLETE、指标 PENDING、对外结论 NOT VERIFIED，直至有真实 artifact。
- 不在 holdout/外部测试集上调阈值、后处理或架构；测试前冻结协议、脚本、模型、阈值、配置。
- 负结果（失败运行、泄漏风险、指标不一致）必须保留并归档，不删除。
- 证据不足时不得声称"分割更准/更鲁棒/可临床使用/SOTA"，改用"当前证据仅支持……仍需验证"。

## 6. 优先查看的文件

1. `docs/records/PROJECT.md`：当前项目事实、任务类型、数据与目标。
2. `docs/records/TASK_BRIEF.md`：本次任务的具体范围与约束。
3. `docs/rules/RESEARCH_RULES.md`：科研纪律与冻结测试规则。
4. `docs/records/EVIDENCE.md` 与 `docs/records/EXPERIMENT_LOG.md`：已有证据与运行记录。
5. `data/README.md`：数据划分、标注口径、路径约定。
6. `CLAUDE.md`：Agent 行为规则。

## 7. 典型输出

- 预测掩膜文件（与输入对齐，默认不入 git，仅以 manifest+checksum 记录）。
- 逐 case 指标表与 aggregate 汇总表（含主/辅指标，取值 TBD/PENDING）。
- 可视化对比图（输入/标注/预测叠加，相对路径引用）。
- 评估运行 manifest：commit hash、环境、seed、数据划分、命令、checkpoint 路径、生成时间。
- 实验记录条目，标注证据等级与状态（PENDING/NOT VERIFIED/INCOMPLETE）。
