# data/ 数据目录约定

> 本文件描述本项目数据目录的组织与登记规则。本模板于 2026-06-19 创建，**当前目录内没有任何真实数据**，下文所有具体字段均为占位符（尖括号）或标记为 TBD。

本目录用于存放与本项目相关的数据集说明、manifest 与 checksum 文件。数据本体（原始文件、影像、特征、缓存）默认**不进入 git**。

---

## 1. git 纳入规则

| 内容类型 | 是否入 git | 替代方案 |
| --- | --- | --- |
| 数据集说明文档（本 README、各数据集登记表） | 入 git | —— |
| manifest 清单（文件列表 + 路径 + 大小） | 入 git | —— |
| checksum 文件（如 SHA-256） | 入 git | —— |
| 原始数据文件 | 不入 git | manifest + checksum |
| 医学影像、概率图、体数据等大体积文件 | 不入 git | manifest + checksum |
| 预处理后的特征/缓存/中间张量 | 不入 git | manifest + checksum，记录生成命令 |

原则：

- **大体积数据与医学影像默认不入 git**；改用 manifest + checksum 记录其存在、位置与完整性。
- 不在 git 中提交隐私路径、受限访问凭据、token 或可定位到受试者的标识。
- 数据根目录路径用占位符 `<数据路径>` 表达，真实路径写入本地配置或不纳入版本控制的文件。

每新增一个数据集，必须在本目录创建对应登记记录（建议每数据集一个文件，例如 `dataset_<数据集名称>.md`），至少包含下列字段。信息未知一律写 TBD，不得猜测。

| 字段 | 说明 | 占位/示例 |
| --- | --- | --- |
| 数据集名称 | 唯一标识 | `<数据集名称>` |
| 来源 | 出处、机构、下载/采集渠道 | TBD |
| 许可或访问限制 | 许可证类型 / 数据使用协议 / 受限说明 | TBD |
| 样本数 | 总样本量（未核对前不得填写估计值） | TBD |
| 模态 | 如影像/文本/表格/信号等 | `<模态>` / TBD |
| 标注类型 | 如分割掩膜/边界框/类别标签/关键点等 | TBD |
| train / validation / test 划分 | 各子集样本数与划分依据；是否含 holdout / 外部测试集 | TBD |
| 预处理 | 处理步骤、生成脚本、生成命令 | TBD |
| 已知标注局限 | 标注噪声、不一致、覆盖不全等 | TBD |
| 排除样本 | 被排除的样本及排除理由（不可静默删除） | TBD |
| 数据根目录 | 本地路径 | `<数据路径>` / TBD |
| manifest 路径 | 该数据集 manifest 文件位置 | TBD |
| checksum 路径 | 该数据集 checksum 文件位置 | TBD |

## 3. manifest 与 checksum

- **manifest**：记录数据集包含的文件清单（相对路径、大小、必要的字段），用于核对数据是否完整、是否与登记一致。
- **checksum**：记录每个文件的哈希值（如 SHA-256），用于完整性校验与跨机器一致性核对。
- manifest 与 checksum 应可由脚本重新生成；生成命令记录在 `docs/EXPERIMENT_LOG.md`。
- 数据完整性核对结果属于可追溯证据，引用 `docs/EVIDENCE.md` 的统一证据等级。

---

## 4. 数据质量纪律

- 在完成数据检查（完整性、划分正确性、标注抽查、泄漏排查）之前，**不得声称数据质量可靠**。
- 划分一旦用于 holdout / 外部测试，须冻结；不在测试集上反复优化方法、阈值或架构。任何潜在数据泄漏必须记录，不可删除。
- 负面发现（标注错误、划分泄漏、样本污染、分布偏移）必须保留并归档，可标注 NOT VERIFIED / PENDING，但不可静默移除。

---

## 5. 证据等级引用

数据相关结论引用 `docs/EVIDENCE.md` 中的统一证据等级（1 假设 → 9 独立复现）。例如"数据划分无泄漏"在仅完成实现层检查时，最多记为等级 3（实现正确性检查），不得描述为更高等级。
