# RESEARCH_LOOP.md —— 迭代研究循环协议

> 本文件定义本科研模板的**迭代闭环**：从**深度研究**出发，沿"计划 → 执行 → 结果 → 报告 → 判断"反复迭代直到达成科研目标，再进入"初稿 → 审稿 → 修订"的写作迭代，直到论文达标。
> **薄控制层**：循环只管阶段调度、迭代与收敛判定、状态留痕；每阶段重活**委托已装 ARS 技能与本仓库 `scripts/`**，绝不重造其流水线。
> 终点 = **一篇满足科研目标与成功判据的学术论文**。
> 本文件是控制协议，不记录项目事实（`PROJECT.md`）/ 证据账本（`EVIDENCE.md`）/ 逐次流水（`EXPERIMENT_LOG.md`）。本模板内不含真实结果。

---

## 0. 全局总览：两个串联迭代环

整个工作流 = **两个串联的迭代环**，中间由一道"报告是否达到科研目标"的关卡分隔。每一轮都是一次**迭代**：做一步 → 判断 → 不达标就回上一关再来一轮，达标才前进。

```text
环 A · 研究迭代环（产生证据）
   ① 深度研究 deep-research
         │
         ▼
   ② 设计 / 更新研究计划  ◄───────────────┐
         │                                │
         ▼                                │  否：据报告暴露的 gap 更新计划，再迭代一轮
   ③ 执行：写码 · 测数据 · 运行 · 检查      │
         │                                │
         ▼                                │
   ④ 结果（artifact + 机械化指标）          │
         │                                │
         ▼                                │
   ⑤ 形成研究报告                          │
         │                                │
         ▼                                │
   ⑥ 报告达到科研目标？ ───── 否 ──────────┘
         │ 是（须人审确认）
         ▼
环 B · 写作迭代环（产出论文）
   ⑦ 写论文初稿（用 Workflow 分章并行）
         │
         ▼
   ⑧ 审稿人评审 · 提出问题清单  ◄─────────┐
         │                                │
         ▼                                │  未达标且未到最大审稿轮次
   ⑨ 据问题修订（＋引用核查）  ────────────┘
         │ 达标（无 critical）或 触发终止条件
         ▼
   ⑩ 定稿（人审确认）── 完成
```

> 同一 agent 在各阶段轮流担任 **执行 / 检查 / 修订** 三职责；"检查"优先用**独立评审**（对候选随机标号、盲化去偏），而非执行者自评"看起来不错"。

## 1. 环 A · 研究迭代环（迭代直到证据达标）

目标：让核心 claim 的**实际证据等级 ≥ 目标证据等级**（1–9，见 `EVIDENCE.md`），形成一份达到科研目标的研究报告。

| 阶段 | 做什么 | 委托 / 工具 | 落点 |
|---|---|---|---|
| ① 深度研究 | 摸清现状、gap、可行性，**据此提出 / 确认科研目标与成功判据**（人审关卡） | `deep-research`（lit-review / systematic-review / three-way-scan） | `PAPER_NOTES.md`、`EVIDENCE.md` |
| ② 设计 / 更新研究计划 | 把目标拆成可执行步骤：实验设计、数据、指标、**实验条件覆盖清单**、预算 | 自拟（**Workflow** 编排）；参 `configs/task_types/<任务类型>.md` | `TASK_BRIEF.md`、本文件 §4 |
| ③ 执行 | **写代码 · 准备 / 测试数据 · 运行 · 检查**（每轮聚焦一处，见下「执行细则」） | `scripts/` + 自有技能 | `EXPERIMENT_LOG.md`、`outputs/` |
| ④ 结果 | 收集 artifact 与**机械化指标**（固定评估脚本输出的数值） | 固定评估脚本 | `EXPERIMENT_LOG.md`、`EVIDENCE.md` |
| ⑤ 形成研究报告 | 把本轮结果汇成结构化报告（结论 ＋ 支撑证据 ＋ 局限） | `deep-research`（report）/ 自拟 | 报告草稿、`EVIDENCE.md` |
| ⑥ 判断 | **报告是否达到科研目标？**（出环判据见 §3，须独立核 ＋ 人审） | `academic-paper-reviewer` ＋ 人审 | 本文件 §4 |

**⑥ 的两个去向（这就是研究迭代）**：

- **未达标 → 回 ②**：据报告暴露的 gap **更新研究计划**，再走 ③→④→⑤→⑥。如此反复。
- **达标 → 出环 A**，进入环 B。**须人审确认达标**。

**执行细则（③，复用现有规则、不重复造）**：每轮**改一处** → **小批量先验**（CLAUDE.md 第 14.3 节）→ 运行（算力高效 第 14.1 节 / 长任务设哨兵 第 14.2 节，**先 commit 再跑**）→ 测试取机械化指标 → **keep / discard**（变差 / 崩溃就 `git revert`，失败零成本）→ 留痕并**升级 claim 证据等级**。配套动作：

- **假设驱动调试**：结果"看着不对"时，把疑因当**可证伪假设**（每轮一条 ＋ 具名技术：二分 / 差分-ablation / 最小复现 / `git bisect` / 溯因），勿乱改污染有效性。
- **回归统计检验**：声称"提速 / 无精度损失"前用 $U$ 检验 ＋ 效应量 ＋ 噪声带（CLAUDE.md 第 8 节）；未真实采样标 `NOT VERIFIED`，不得虚构。
- **实验条件覆盖**：枚举 基线 / 消融 / seed 方差 / 规模 / OOD / 泄漏控制 / 失败-恢复 / 算力，连续 ≥ 3 轮无新条件即**饱和**。

## 2. 环 B · 写作迭代环（迭代直到评审达标）

进入条件：环 A 报告经**人审确认达到科研目标**。

| 阶段 | 做什么 | 委托 / 工具 | 落点 |
|---|---|---|---|
| ⑦ 写初稿 | 据已验证证据写**论文第一版**，**尽量用 Workflow 分章并行** | `academic-paper`（`/ars-outline` → `/ars-full`）＋ Workflow | 稿件、`PAPER_NOTES.md` |
| ⑧ 审稿 | 设独立**审稿人**多视角评审，**提出问题清单**（critical / major / minor） | `academic-paper-reviewer`（`/ars-reviewer`，含 Devil's Advocate，盲化去偏） | `EXPERIMENT_LOG.md` |
| ⑨ 修订 | 据问题清单逐条修订、回填证据、引用核查 | `academic-paper`（`/ars-revision`）＋ `/ars-citation-check` | 稿件、`PAPER_NOTES.md` |

**⑧ ⇄ ⑨ 反复迭代**：审稿 → 修订 → 再审稿，质量逐轮抬升，直到 **⑧ 评审无 critical（达标）** 或触发**终止条件**（见 §3）。达标后 **⑩ 人审定稿**。

## 3. 迭代上限 · 停止 / 终止条件 · 人审关卡

**环 A 出环（达标）须同时满足**：核心 claim 实际证据等级 ≥ 目标等级；报告结论均可追溯；负结果 / 局限如实呈现；**人审确认**（关卡 ⑥）。

**环 A 终止（防死循环）**：剩余工作量（见 §4）连续 ≥ 3 轮不降 → **PLATEAU**；关键证据不可得 → **BLOCKED**；达**最大轮次（默认 ≤ 8 轮，可设）** → 强制停、如实上报，**不得降标收尾**。

**环 B 出环（达标）**：最近一轮 `academic-paper-reviewer` **无 critical**（major 已处理或有据保留）、引用 / 映射无误、局限完整、**人审定稿**（关卡 ⑩）。

**环 B 终止**：达**最大审稿轮次（默认 ≤ 5 轮，可设）** 或连续 2 轮无实质改进 → 停、上报当前稿与未决问题。

**必须人审的关卡**：① 科研目标设定 / 重大变更（增删核心 claim、改目标证据等级、换目标期刊）；⑥ 报告达标判定（出环 A）；⑩ 论文定稿。agent 均**不得自行拍板**。

**反滥用闸门**：达不到目标证据等级时**不得在环内私自降 claim 强度 / 降目标等级**（须走"重大变更"人审）；目标证据等级须匹配结论强度（强结论 ≥ 7，临床 / 泛化类须外部或独立证据，RESEARCH_RULES 第 11 节）；统计量未真实采样标 `NOT VERIFIED` 不得虚构；证据不足的 claim 即使"读着完整"仍 `PENDING`；缺证据不得编造补齐（RESEARCH_RULES 第 1 节）。
