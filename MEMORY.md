# MEMORY.md —— 长期经验记忆索引

> **这不是临时笔记区。** 本文件只记录**长期、跨多次实验仍然成立、对未来决策有指导价值**的经验。
> 临时记录（单次运行、临时调试、当天进展、未沉淀的观察）一律放 `docs/records/EXPERIMENT_LOG.md`，不要写在这里。
>
> 本文件是**空模板**：除"本模板于 2026-06-19 创建"这一事实外，不含任何真实项目经验。各分类当前为空，仅给占位与写入须知。

## 1. 写入须知（每次新增前先读）

- **只记长期有价值的信息**：能反复指导未来工作的结论、约束、决策；一次性的东西不写这里。
- **可追溯**：涉及结论的条目，尽量链接到支持它的 artifact / 实验记录 / commit hash，并标注证据等级（见下）。
- **不编造**：无真实 artifact 支持的结论标 PENDING / NOT VERIFIED；未知值标 TBD；不写虚假指标、数据规模、对比或结论。
- **负结果优先保留**：已被否定的假设、已知失败模式属于高价值长期经验，应登记而非丢弃。
- **谨慎表达**：证据不足时不得声称泛化强 / 鲁棒 / SOTA / 临床可用等；用"当前证据仅支持……/ 仍需验证"等措辞。
- **追加而非覆盖**：经验更新时保留旧结论与其失效原因，便于回溯。

### 统一证据等级（全模板共用）

1 假设 → 2 设计方案 → 3 实现正确性检查 → 4 开发集验证 → 5 内部验证 → 6 冻结内部测试 → 7 锁定 holdout 测试 → 8 外部验证 → 9 独立复现。（完整定义以 `docs/rules/RESEARCH_RULES.md` 第 0 节为准。）

> 不得把低等级证据描述成高等级。

### 单条经验建议字段

- **结论 / 经验**：……
- **适用范围**：……（哪些任务类型 / 数据 / 环境）
- **支持证据**：`<artifact 路径/ID>` + 证据等级 + commit hash（或标 PENDING / NOT VERIFIED / TBD）
- **登记日期**：YYYY-MM-DD
